<p><b><i>Applies to </i></b><a href="1e855f94-4617-47e4-b89e-0856c6cb420f.html"><b><i>RDL 2008/01</i></b></a><b><i>,
</i></b><a href="3428e690-a348-4ec7-8a6a-8efb42d2cdee.html"><b><i>RDL 2010/01</i></b></a><b><i>,
and </i></b><a href="52ce3983-2bfc-4e72-9359-42aaf5fe4509.html"><b><i>RDL 2016/01</i></b></a></p>

<p>The <b>ChartLegend.DockToChartArea</b> element specifies the
name of the <a href="74e08a7c-5405-4ea4-b903-a79ef4d215f7.html">ChartArea</a>
on which to draw a <a href="68a0757c-8f1a-42b9-9473-ccedd40029fb.html">ChartLegend</a>.
The <b>ChartLegend.DockToChartArea</b> element is optional. If the <b>ChartLegend.DockToChartArea</b>
element is present, its value MUST be a <a href="1ed81ef3-a683-45e3-aaad-bd2bbe71bc3d.html">String</a> (<a href="https://go.microsoft.com/fwlink/?LinkId=90610">[XMLSCHEMA2]</a> section
3.2.1). If this element is not specified or if it does not match any <b>ChartArea</b>
name, the legend MUST be drawn relative to the chart area.</p>
