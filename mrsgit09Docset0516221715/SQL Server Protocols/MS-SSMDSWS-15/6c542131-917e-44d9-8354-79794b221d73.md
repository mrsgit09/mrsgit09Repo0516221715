<p><b>CodeGenerationSeed</b>: Specifies the starting
value from which new member codes are to be generated. This value is ignored
unless <b>IsCodeGenerationEnabled</b> is set to &quot;true&quot;.</p>

<p><b>DataCompressionType</b>: Specifies the type of the
data compression.</p>

<p><b>Description</b>: Specifies the string that
represents the long description of the entity.</p>

<p><b>ExplicitHierarchies</b>: Specifies the list of the
explicit hierarchies.</p>

<p><a id="OLE_LINK2"></a><a id="OLE_LINK1"><b>Indexes</b></a>:
Specifies the custom user indexes on the entity.</p>

<p><b>IsBase</b>: Specifies whether an entity is a base
entity.</p>

<p><b>IsCodeGenerationEnabled</b>: Specifies whether
automatic code generation is enabled. When set to &quot;true&quot;, new members
are created with an autogenerated, numeric, incrementing code value that starts
at <b>CodeGenerationSeed</b>. When &quot;false&quot;, the member creator
specifies a code value.</p>

<p><b>IsCollectionEnabled</b>: Specifies whether the
entity is enabled for collection operations.</p>

<p><b>IsFlat</b>: Specifies whether the entity supports
explicit hierarchies.</p>

<p><b>IsHierarchyEnabled</b>: Specifies whether the
entity is enabled for explicit hierarchy operations.</p>

<p><b>IsSyncTarget:</b> Specifies whether this entity is
the target of a sync relationship.</p>

<p><b>IsSystem</b>: Specifies whether an entity was
created by the MDS system. System entities cannot be deleted.</p>

<p><b>MemberTypes</b>: Specifies the list of the types
of the entity members.</p>

<p><b>RequireApproval</b>: Specifies whether the entity
requires approval.</p>

<p><b>StagingName</b>: Specifies the staging name for
the entity.</p>

<p><b>TransactionLogType</b>: Specifies the transaction
log type.</p>
